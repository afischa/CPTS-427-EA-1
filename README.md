# CPTS-427-EA-1
For this project I did the crimson code 2026 hackathon. We had to build an app with the theme "reinventing the wheel”. I was in a group of 4 and the app we landed on making was a travel information guide app. The user will point their phone in the direction of a building or a monument and a paragraph will pop up giving them information about it. We felt like this was good for the theme because it was like your normal information finder but it does not require you to know what the place is called. Also you do not need to do searching, it just pops up the info. 

We first started out by making a component diagram of the project for what we hoped to accomplish and how each thing will interact. This helped us lay out the plan and gave us an agreement on what the flow was going to be. This also allowed us to delegate tasks to each person so they would know what part they would make. 
	
The task I was assigned was to collect the places that were around the user and figure out which one they are facing and have it pop up on the screen. I did this by using an API call to get wiki articles from wiki maps in a certain radius of users longitude and latitude. Then using the compass I get the direction the user is facing and figure out what place is closest to that direction. 
	
I think I contributed well to my group. I think something I would change if we could redo the event is to get the project up and going faster. We had some trouble and ran into issues getting our repo set up and configured which created a bottleneck where only one person could realistically work on setting up the project causing the rest of us only to be doing research which can only get you so far. This caused us not to be able to implement extra features to our project that we wanted to add.
